# QA Plan – File Upload Validator

## Test Levels
- Unit Testing: Tests each validation rule such as file type, file size, and filename checks.
- Integration Testing: Confirms that validation rules work together correctly in the upload process.
- System Testing: Verifies the whole validator behaves as expected based on requirements.

## Entry Criteria
- Source code is complete.
- Test environment is set up.
- Test cases are prepared.

## Exit Criteria
- All planned tests are executed.
- Failed tests are investigated.
- High severity defects are fixed.
- Final results are documented.

## Severity Levels
- S1 – Critical: Major failure or security risk.
- S2 – High: Important feature does not work correctly.
- S3 – Medium: Issue affects expected behavior but has workaround.
- S4 – Low: Minor issue with minimal impact.
