# Support Plan – File Upload Validator

## Support Overview
This document defines how issues will be reported, handled, and escalated after deployment.

## Issue Reporting Process
Issues should be reported through:
- GitHub Issues
- direct message to the assigned support owner
- screenshots and steps to reproduce, when possible

## Common Issues
- Unsupported file type submitted
- Empty filename entered
- File exceeds size limit
- App fails to load online
- Warning message not appearing correctly

## Response Times
- Critical issues: within 4 hours
- High priority issues: within 12 hours
- Medium priority issues: within 24 hours
- Low priority issues: within 48 hours

## Escalation Process
1. User or tester reports issue
2. QA Lead verifies the issue
3. Developer investigates root cause
4. DevOps Lead checks deployment if issue is environment-related
5. Fix is tested before redeployment

## Support Roles
- QA Lead: validates reported issues
- Developer: fixes source code issues
- DevOps Lead: handles deployment/platform issues
- Docs Lead: updates documentation if process changes
